Built for a cleaner comeback

Neko-Love now has a setup that feels ready to ship: local assets, filter routes, OpenAPI 3.1 docs, a static front page, and a documentation experience that looks closer to a real product than a raw developer tool.

OpenAPI 3.1 Ready for docs tooling and SDK generation
18 filters Including animated GIF support
Pages-ready Static site separated from the Go API

Deployment split

Host this static folder on GitHub Pages and keep the Go API on a VPS, Fly.io, Render, Railway, or your own server. That way you get a clean public site without trying to force dynamic Go code onto static hosting.

Set CORS_ALLOW_ORIGINS on the API to your Pages domain if you want live “Try it out” requests from the static docs.

Examples

Quick usage snippets

cURL

curl "https://api.neko-love.com/api/v4/neko"

curl "https://api.neko-love.com/api/v4/filters/pixelate?image=https://example.com/cat.png&pixel_size=12"

JavaScript

const meta = await fetch("https://api.neko-love.com/api/v4/neko")
  .then((res) => res.json())

const imgUrl = new URL("https://api.neko-love.com/api/v4/filters/pixelate")
imgUrl.searchParams.set("image", "https://example.com/cat.png")
imgUrl.searchParams.set("pixel_size", "12")

Python

import requests

meta = requests.get("https://api.neko-love.com/api/v4/neko").json()

img = requests.get(
    "https://api.neko-love.com/api/v4/filters/pixelate",
    params={"image": "https://example.com/cat.png", "pixel_size": 12},
)
